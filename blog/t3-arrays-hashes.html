Arrays


According to Tutorialspoint.com arrays "are ordered, integer-indexed collections of any object. Each element in an array is associated with and referred to by an index." Say what? We'll we can take a look at what one looks like:
array = [0,1,2,3] or varname = array["a", "b", "c"]

It is a way to point at certain things, be it a string, an integer, or a float within a contained index. Like other languages, it starts counting at 0. So the first number above, which is 0, is position 0 in the array, then 1, then 2, and so on. The first one above ends at 3. You could also refer to it as '-1' since if you skip back to the end from zero, it would jump to the 3. 'A' in the second array is position 0, 'B' is 1, 'C' is 2 OR -1. It depends on how you are setting up your code with how you will refer to it .

Hashes


Now what is a hash, you ask? Let's go back to our friends at Tutorialspoint.com. They say:
"A Hash is a collection of key-value pairs like this: "employee" => "salary". It is similar to an Array, except that indexing is done via arbitrary keys of any object type, not an integer index.

See that difference? Did you spot it? Do you get it? If you said yes, pat yourself on the back. Hashes are indexed by words, phrases, or as they say 'keys.' They are not referenced by integers. So you do not point at it via 0, 1, 2 or 3. A hash can look like this:
awesome_points = { "Lesley" => "a million", "Judy Garland" => "so much more" }

You can also type to make a key ' :lesley ' or ':judy_garland' or ':person_anything.' You can use the syntax 'do' to print them or other various things. Say you have a hash of items and prices, you can list them out. There are various ways to create hashes, and you can pick a favorite. There are {} hashes. There are variable_name = Hash.new, followed by variable_name[1] = "blahblah" variable_name[2] = "ewwwww", and so on. It can depend on preference or what type of hash or code you are trying to make. It also can depend on how concise you want to be, who may be reading your code, or things such as speed. They are good for database usages and also receiving feedback.
